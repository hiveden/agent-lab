# PRODUCT-TODO: Radar MVP

**定位**: 单用户，多设备端，信息生命周期管理。
**核心**: Web 端重管理与反思，Mobile 端极轻消费与追踪。

## 阶段一：Web 端配置与指挥

### 1. 信息源配置 (Sources)
- **功能**: 管理信息源 (增删改) 及注意力预期。
- **验收**:
  - `sources` 表 CRUD。
  - 配置各 Source 的预期注意力占比 (`expected_weight`)，总和 100%。

### 2. 手动执行与透明化 (Run & Trace)
- **功能**: 手动触发采集，实时查看处理过程。
- **验收**:
  - 提供手动 Run 按钮。
  - SSE 实时渲染执行日志 (Trace Drawer)。
  - 展示漏斗数据：发现总量 -> 过滤量 -> 入库量。
  - 完整运行记录入库 (`runs` 表)。

## 阶段二：Mobile 端消费与追踪

### 3. 流式信息流处理 (Stream & Triage)
- **功能**: 极速处理信息流，无打断。
- **验收**:
  - **流式布局**: 手机侧采用流式布局 (无限瀑布流/标准列表)。
  - 分区展示：“待处理”与“已处理”双视图。
  - 快捷操作：支持左滑/右滑或底部大按钮决定保留/丢弃。
  - 乐观 UI：操作后卡片立即移除。

### 4. 隐式行为追踪 (Implicit Tracking)
- **功能**: 基于真实行为记录注意力。
- **验收**:
  - **状态自动跃迁**: 点击卡片静默变更为 `viewing`。
  - **停留时长统计**: 详情页挂载即计时，结合 Page Visibility 排除后台时间，卸载时累加上报 `view_duration_ms`。

### 5. 轻量级对话与追问 (Mobile Chat UI)
- **功能**: 将详情阅读转化为沉浸式对话交互。
- **验收**:
  - **交互样式**: 点击卡片直接进入对话框交互样式 (顶部为摘要，主体为消息流)。
  - **键盘适配**: 处理输入法键盘唤起问题 (视口自适应、输入框吸底、消息流自动滚动到底部防遮挡)。
  - **状态跃迁**: 发送首条消息，条目状态静默升级为 `discussing`。
  - **权重计算**: 对话轮次作为最高注意力权重指标记录。

## 阶段三：Web 端周期性反思

### 6. 偏差分析报告 (Weekly Reflection)
- **功能**: 对比预期与实际注意力消耗，生成认知报告。
- **验收**:
  - **数据聚合**: 定时统计本周各项 `view_duration_ms`、对话轮次及负向操作，计算实际注意力分布。
  - **LLM 分析**: 将实际分布与配置的 `expected_weight` 对比，生成偏差洞察报告。
  - **数据可视化**: Web 端生成专属面板展示图表与文字报告。

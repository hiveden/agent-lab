# Radar 产品设计哲学

**核心原则：不要废话，直击本质。**

## 1. 核心定位：认知之镜 (Cognitive Mirror)
Radar 不是另一个 RSS 阅读器，也不是单纯的 AI 聊天工具。
它的本质是一面映射真实注意力的“认知之镜”。

## 2. 终极交付物：每周回顾报告
产品对用户的最终交付是一份“自我反思报告”。
核心逻辑是揭示数据上的 GAP（认知落差）：
- **理想自我 (Ideal Self)：** 用户配置的订阅源、Prompt 设定、自以为关心的领域。
- **真实自我 (Actual Self)：** 用户身体力行投入注意力的领域。
通过对比两者，让用户认清真实的自己。

## 3. UI/UX 原则之一：效率至上与心流 (Efficiency & Flow)
- **终端分工明确：** 移动端绝对专注于“轻消费”（滑动过滤、碎片化阅读）；桌面端作为“指挥中心（Command Center）”处理深度交互和系统配置。
- **零打断：** 严禁使用强制性反馈弹窗（如“评价本文”）打断用户的阅读心流。

## 4. UI/UX 原则之二：行为即数据 (Interaction is Signal)
全面弃用显性的打分系统，转用隐性数据静默构建用户的“注意力画像”。
用户的每一次无意识交互，都是系统收集的信号：
- 滑动/丢弃 (Swipe/Skip) -> 负向/弱信号
- 页面停留时长 (Dwell time) -> 正向兴趣信号
- 与 AI 的对话轮次与深度 (Chat rounds/depth) -> 极强兴趣信号
